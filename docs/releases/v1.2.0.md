# MyDesk v1.2.0

MyDesk v1.2.0 is a minor release focused on turning the canvas into a more dependable visual workflow layer above an existing file system. It improves connection routing, resizing, zoom behavior, snippet workflows, storage stability, and product documentation.

## Product Direction

MyDesk is built for users who already maintain rigorous project folders, naming rules, and local archives, but need a clearer way to reuse the same files across multiple project contexts.

This release strengthens the core idea: keep real files where they are, then organize their project-specific meaning through visual cards, notes, frames, snippets, and connections.

## New and Improved

- Added obstacle-aware canvas connection routing so links can automatically avoid cards during layout, link creation, and card movement.
- Improved link anchor behavior for cards, files, folders, note cards, and organization frames.
- Added support for routed links that still respect manually placed connection control points.
- Added lock/unlock behavior for link anchors and expanded control-point regression coverage.
- Reworked card and organization-frame resize handles so resizing no longer conflicts with card dragging or canvas panning.
- Improved zoom scaling so canvas content behaves more like a visual board, with card elements scaling together.
- Added Settings support for canvas scroll-zoom direction and default zoom baseline.
- Added a safer SwiftData storage bootstrap path with clearer startup failure handling.
- Added app-specific persistent store layout and backup retention tests.
- Expanded Snippet Library and workspace snippet behavior with editing, deletion, copying, and expanded viewing.
- Updated Home and library surfaces for better access to resources and snippets.
- Expanded `docs/feature-checklist.md` as a practical regression checklist for future development.
- Rewrote the README with bilingual navigation, product positioning, pain-point framing, install instructions, source usage, and release package guidance.

## Fixes

- Fixed resize handles becoming unresponsive because parent card gestures could intercept resize drags.
- Fixed connection lines failing to avoid cards after cards were moved.
- Fixed routed lines with manual control points not avoiding obstacles correctly.
- Reduced cases where arrows or animated flow segments appeared inside or behind card content.
- Improved hit testing around scaled cards and frame borders.
- Improved SwiftData crash resilience around store startup and migration-like failures.
